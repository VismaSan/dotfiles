# dotfiles

Personal terminal environment: zsh + oh-my-zsh + powerlevel10k, tmux (+TPM), Neovim,
[herdr](https://herdr.dev), Homebrew-managed CLI tools, and Claude Code settings.

Works on **macOS and Linux**. One command sets up a bare machine, and the same
command is safe to re-run any time to bring an existing machine up to date ‚Äî
this isn't just a fresh-install script.

## Usage

```sh
git clone <this-repo-url> ~/dotfiles   # or clone it anywhere you like
cd ~/dotfiles
./install.sh
```

Re-run `./install.sh` whenever you want to pick up updates (newer Homebrew
formulae, latest powerlevel10k/TPM/nvim commits, etc.) ‚Äî every step installs
what's missing and updates what's already there.

## Prerequisites

- macOS or Linux. (Windows/WSL is untested ‚Äî treat as Linux at your own risk.)

`install.sh` clones `VismaSan/nvim` over plain HTTPS
(`scripts/07_nvim.sh`), so no SSH key or GitHub auth is required to run it
as configured today. If that repo's clone access ever changes such that
HTTPS stops working, switch the URL there back to
`git@github.com:VismaSan/nvim.git` and you'll need an SSH key registered
with GitHub on the new machine before running `install.sh`.

## What gets set up

| Piece | How |
|---|---|
| Homebrew | official installer if missing, `brew update` if present |
| CLI tools | `brew bundle --file=Brewfile --upgrade` (see `Brewfile`) |
| oh-my-zsh + powerlevel10k | official installer / `git pull` on the p10k theme repo |
| `.zshrc`, `.p10k.zsh`, `.tmux.conf` | symlinked from this repo into `$HOME` |
| Claude Code settings | `settings.json`, `skills/`, `statusline-command.sh`, plugin config symlinked into `~/.claude` ‚Äî **runs before herdr**, see below |
| herdr | official installer (self-updating), plus `herdr integration install claude`; `config.toml` symlinked into `~/.config/herdr` |
| tmux plugins (TPM, dracula/tmux) | TPM cloned/pulled, `install_plugins` + `update_plugins all` |
| Neovim config | cloned/pulled from `VismaSan/nvim`, `lazy.nvim` synced headlessly |
| .NET SDK + global tools | cross-platform `dotnet-install.sh`, then `dotnet-format`/`csharpier` |
| Nerd Font | Homebrew cask on macOS, downloaded + `fc-cache`'d on Linux |
| Default shell | `chsh` to the Homebrew-installed `zsh`, skipped if already on any zsh |

## The one manual step

Set your terminal app's font to **"Hack Nerd Font"** in its preferences ‚Äî
there's no scriptable, cross-platform way to do this from a terminal, since
it's a GUI setting per terminal emulator. The font file itself is installed
automatically (see table above); you just need to select it.

## OS-specific notes

- **Linux:** there's no scripted terminal-emulator install ‚Äî iTerm2 (used on
  macOS) has no Linux equivalent, so use whatever your distro/desktop provides.
- **macOS:** iTerm2 is installed via Homebrew cask (`cask "iterm2"`).

## Known companion fix (separate repo)

`VismaSan/nvim`'s `lua/visma/core/options.lua` currently hardcodes a stale
Linux path (`/home/vsanda/.dotnet/tools`) onto `PATH`. This should be fixed
in the nvim repo itself (not here) to something portable, e.g.:

```lua
vim.env.PATH = vim.env.PATH .. ":" .. vim.env.HOME .. "/.dotnet/tools"
```

## What's intentionally excluded

- `~/.claude.json`, `~/.claude/security/`, `~/.claude/session-env/`,
  `~/.claude/history.jsonl`, `sessions/`, `projects/`, and other cache/session
  state ‚Äî machine-local or contains auth tokens, never symlinked or committed
  (see `.gitignore` and `scripts/07_claude.sh`, which only symlinks specific
  known-safe leaf files).
- `~/.config/nvim-safed` ‚Äî an inactive, separate Neovim config, out of scope.
- `~/.claude/plugins/marketplaces/` ‚Äî fetched cache of each marketplace's
  upstream repo (hundreds of files, managed by Claude Code itself); rebuilt
  automatically from `settings.json`'s `extraKnownMarketplaces` +
  `enabledPlugins`, so it's never symlinked or committed here.
- `go`, `cloudflared`, `pgpdump`, `postgresql@14`, `starship`, `mkcert`,
  `supabase` ‚Äî none of these are used by anything else in this repo
  (powerlevel10k, not starship, is the active prompt; nothing here needs a
  local Postgres/pgpdump/cloudflared/Go toolchain or locally-trusted dev
  certs). Install manually if a given machine needs one. `node` stays in the
  Brewfile despite being otherwise unused day-to-day, since Mason (used by
  the nvim config) needs it to install its Node-based LSP servers ‚Äî
  `ts_ls`, `eslint`, `jsonls`, `dockerls`.
- `~/.config/herdr/{*.log,*.sock,session.json}` ‚Äî runtime logs, IPC sockets,
  and workspace/session state, all machine-local. Only `config.toml`
  (keybindings/theme) is portable config and gets symlinked.
- `~/.claude/hooks/herdr-agent-state.sh` ‚Äî installed and fully regenerated by
  `herdr integration install claude` itself (its own header says as much:
  "reinstalling or updating the integration overwrites this file"). Not
  tracked here, same reasoning as `plugins/marketplaces/`. This is also why
  `scripts/04_claude.sh` runs *before* `scripts/05_herdr.sh` ‚Äî herdr's
  installer patches `~/.claude/settings.json`'s `hooks` key with a path
  correct for the current machine, and since `settings.json` is symlinked
  into this repo, whichever step runs last wins; herdr needs to be last so a
  stale, other-machine path never gets committed.

Note: herdr's configured prefix key (`ctrl+s`, in `herdr/config.toml`) is the
same as tmux's prefix (`set -g prefix C-s` in `tmux/tmux.conf`) ‚Äî worth
knowing if you use both at once, since one will shadow the other depending on
which has focus.
